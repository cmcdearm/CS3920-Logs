Logs are important as they are basically a report of every thing that has been done. Logs enable the following to be done easily;
### Troubleshooting and Debugging
- Logs can be used to find out what went wrong within an application or system.
- Logs can be pieced together to reproduce an error or warning.
### Security Monitoring and Threat Analysis
- Logs record privilege changes, system events, authentication attempts, and errors.
- These monitoring capabilities can be used for detecting malicious behavior such as unauthorized access, lateral movement, and brute-forcing.
### Auditing
- Logs create a known-good record of what has happened. This is useful for knowing what events occurred in the leadup to an incident or knowing if an incident is about to occur.
### System Performance Insights
- Logs can be used to see hardware usage with certain applications.
- They give insight into how different programs or events affect our system resources.
### Legal Evidence
- Logs show what occurred in a system at a time, this can be used as legal evidence in an investigation in the event of a breach.
- More critical logs may be required to be kept by some laws.
### Ease of History
- Logs allow you to see what happened when and what triggered it. It allows you to see what has happened for a system to get into a state.
